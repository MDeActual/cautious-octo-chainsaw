# TRUST_CLAIMS_POLICY.md

## SecurePulse — Trust Claims Policy

**Owner:** CloudMatrix Business Solutions  
**Applies to:** All marketing copy, product UI, sales materials, documentation, and AI outputs  
**Source authority:** `GITHUB_DEVOS_OPERATOR_v2_SECUREPULSE.md` Trust Claim Rules  
**Last updated:** May 2026

---

## Purpose

This policy defines what SecurePulse and CloudMatrix may and may not claim in any public or customer-facing context. The goal is to protect customers from misleading claims and protect CloudMatrix from regulatory, legal, and reputational risk.

If a claim cannot be verified by evidence in this repository or by a third-party certification, it must not be made.

---

## Approved Language

The following claims are accurate and may be used freely:

### Technology Stack
- ✅ "Built on Microsoft Azure"
- ✅ "Powered by Microsoft Entra ID authentication"
- ✅ "Integrates with Microsoft 365 and Microsoft Graph API"
- ✅ "Uses Microsoft Secure Score as the security assessment foundation"
- ✅ "AI insights powered by Azure OpenAI Service"
- ✅ "Microsoft Cloud Partner technology"

### Compliance and Security Framework Alignment
- ✅ "Aligned with CIS Controls v8"
- ✅ "Supports PIPEDA compliance readiness"
- ✅ "Supports Quebec Law 25 compliance readiness"
- ✅ "Supports Microsoft Zero Trust framework alignment"
- ✅ "Designed for Canadian regulatory contexts"
- ✅ "Maps security controls to CIS Controls v8 safeguards"

Note: Use "supports readiness" or "aligned with" — not "certifies compliance with" or "guarantees compliance."

### Platform Positioning
- ✅ "Microsoft-native security posture platform"
- ✅ "Purpose-built for Canadian SMBs"
- ✅ "MSSP-grade architecture with tenant isolation"
- ✅ "Security-first design with audit logging"
- ✅ "AI-assisted security operations"
- ✅ "Copilot for Microsoft 365 readiness assessment"

### AI Transparency
- ✅ "All AI outputs include model version, guardrails applied, and data sources used"
- ✅ "AI summaries are generated from normalized assessment data, not raw tenant records"
- ✅ "Azure OpenAI content filtering active in production"

## Prohibited Claims

The following claims must NEVER be made without the specified evidence:

| Claim | Why Prohibited | What Is Required to Use It |
|---|---|---|
| "SOC 2 certified" or "SOC 2 compliant" | Not certified | Completed SOC 2 Type II audit by accredited auditor |
| "ISO 27001 certified" | Not certified | Completed ISO 27001 certification |
| "Microsoft-endorsed" or "Approved by Microsoft" | Not endorsed | Formal Microsoft partnership endorsement documentation |
| "Certified Microsoft Security partner" | Not a formal certification | Completion of Microsoft Cloud Security Specialization |
| "Guaranteed breach prevention" | No security product can guarantee this | Remove entirely — no acceptable alternative |
| "100% compliant with PIPEDA" | Compliance depends on customer implementation | Change to "supports PIPEDA compliance readiness" |
| "Prevents data breaches" | Misleading | Change to "reduces security exposure" or "improves security posture" |
| "Reduces risk by X%" | Unverifiable unless explicitly modeled | Add: "estimated based on [model/methodology], not a guarantee" |
| "Government-approved" | Not approved | Remove entirely |
| "Enterprise-grade security" | Vague and unverifiable | Replace with specific claims about architecture features |
| "The most secure platform for SMBs" | Unverifiable superlative | Remove; use specific differentiators instead |

## ROI Disclaimer Requirement

Any ROI or financial benefit claim must include one of the following disclaimers:

**Short form (for UI/dashboard):**
> "Estimated based on industry averages. Actual results may vary."

**Long form (for proposals, reports, sales materials):**
> "ROI estimates are modeled projections based on industry-reported breach costs and remediation effort estimates. They do not represent guaranteed financial outcomes. Actual results will vary based on organizational context, implementation completeness, and external threat factors."

---

## AI Output Claims

All AI-generated content in SecurePulse must be:

1. **Labeled as AI-generated** — never presented as a certified security audit or legal opinion
2. **Transparency metadata included** — `model_version`, `guardrails_applied`, `data_sources` populated in the API response and visible in the AI transparency panel
3. **Scoped to assessment data** — AI prompts must never include raw tenant user data, personally identifiable information, or security incident details beyond aggregated scores

The following label must appear near any AI-generated summary in the UI:
> "This summary was generated by AI and is intended to support — not replace — professional security review."

## Enforcement

- All new marketing copy must be reviewed against this policy before publication
- All AI prompt templates must be reviewed for data minimization compliance
- All product UI strings must avoid prohibited language
- This document must be updated whenever new certifications are obtained or new claims are approved

Violations identified in code review or UI review must be flagged as blocking issues.
